### 1.相关目录与文件

```
cd /usr/local/apache2/      //进入apache 安装目录
ls    //查看
显示
bin   	//存放apache 相关命令的目录
etc		//apache 配置文件所在的目录
htdocs	//网站默认根目录
logs	//日志文件所在的目录 ,访问(access_log)日志,错误(error.log)日志,进程(httpd.pid)日志

配置文件
/usr/local/apache2/etc/httpd.conf     //主配置文件
/usr/local/apache2/etc/extra/http- *.conf  //12个子配置文件

网站默认根目录
/usr/local/apache2/htdocs/

网站日志文件目录
/usr/local/apache2/logs/
日志处理:
logrotate   
原理:日志的切割与轮替

打开系统自带的日志处理文件
vim /etc/logrotate.conf
显示
weekly   //按周切割
rotate   //4次备份(即切割)
create	 //建立空文件
dateext  //后缀名为dateext
include /etc/logrotate.d    //导入系统默认的logroate 这个服务
/var/log/wtmp{
	....
}
/var/log/btmp{
	....
}           //系统默认的日志切割服务

向文件中添加
/usr/local/apache2/logs/access_log {  //访问日志的处理
	 daily		//按天来进行处理
	 rotate 30		//每天建立30个空文件
}
/usr/local/apache2/logs/error_log{     //错误日志的处理
	 daily		//按天来进行处理
	 rotate 30		//每天建立30个空文件
}

//要等到明天才看到切割,现在要立即执行切割,使用
logrotate -f /etc/logrotate.conf
ls 查看 
/usr/local/apache2/logs/ 下多出2个日志文件
```
