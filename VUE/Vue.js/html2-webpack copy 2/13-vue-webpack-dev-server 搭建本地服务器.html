webpack 提供了一个可选的本地开发服务器,这个本地服务器基于node.js 搭建,内部使用express 框架,可以实现我们想要的让浏览器自动刷新显示我们 修改后的结果 不过他是一个单独的模块,在webpack 中使用之前需要安装它 安装插件 npm install --save-dev webpack-dev-server@2.9.3 devServer 也是作为webpack 中的一个选项,选项本身可以设置如下属性 contentBase:哪一个文件提供本地服务,默认是根文件,我们这里写./dist port:端口号 默认8080 端口 inline:页面实时刷新 historyApiFallback:在SAP 页面中,依赖于HTML5 的history 模式 配置里面配置 devServer:{ contentBase:'./dist', inline:true, } 当前文件目录下 终端cmd webpack-dev-server 命令是在全局下跑 中端要用反斜杠 .\node_modules\.bin\webpack-dev-sever.cmd 在局部下跑 json 配置 "dev":"webpack-dev-server" 开发时执行的是这个脚本 "dev":"webpack-dev-server --open" 开发时执行的是这个脚本,自动打开页面 配置完后 执行 npm run dev http://localhost:8080/ 在此页面 保存时自动刷新 写完后 在 npm run build 真正打包,提交到服务器部署
